# Published specs

Canonical `ContractSpec` JSON blobs for every well-known spec that's actually
been published through the live on-chain registry contract, generated by
`scripts/seed-well-known.ts`. Each file is named `<contractId>.json` and is
served from GitHub's raw content at `main` - the on-chain registry's
`pointer` field for that contract points here, and the `spec_hash` it stores
is the sha256 of this file's contents (via `canonicalizeSpec`).

Don't hand-edit these - re-run `pnpm seed:well-known` (with a newer version
in `specs/well-known/*.json`) instead, which publishes a new immutable
version rather than mutating an existing one.
